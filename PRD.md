## 一、项目概述

**堆柴 (DuiChai)** — 运动场地的UGC分享与社交平台。

**品牌含义：** 「堆柴」既是动词也是名词。古语云「众人拾柴火焰高」—— 大家一起来堆柴，柴火越多火焰越高。用户给场地添柴，就是一起把场地点燃、烧旺。

**宣传语：** 众人拾柴火焰高

**域名：** [duichai.com](https://duichai.com) （已注册，将接入 Cloudflare 5号账户）

### 产品定位

运动场地UGC分享平台 + 同好社交

### 目标用户

全品类运动爱好者（篮球、足球、羽毛球、网球、跑步、滑板等）

## 二、UI/UX 设计风格

### 设计语言

**极简化 · 沉浸感 · 动态交互**

在兼顾视觉美感的同时，更注重信息层级的清晰和无障碍体验。

### 核心原则

| 原则 | 说明 |
|------|------|
| 极简化 | 去除冗余装饰，内容即界面，让场地信息成为主角 |
| 沉浸感 | 全屏地图/图片浏览，弱化导航栏干扰，滑动交互相册 |
| 动态交互 | 添柴/置顶等操作有微动效反馈，加载动画统一 |
| 信息层级 | 柴火数 > 场地名 > 地址 > 补充信息，视觉权重明确 |
| 无障碍 | 大字体模式适配，色彩对比度达标，VoiceOver支持 |

### 语义缩放与自适应布局（Semantic Zoom & Adaptive UI）

- **特点：** 根据设备类型、使用习惯或特定场景，界面元素（文字大小、按钮位置、模块分布）智能重排或缩放
- **优势：** 跨设备（手机、平板、手表等）无缝体验
  - 手机：单列流式布局，全屏地图
  - 平板：双列/三列布局，侧边栏常驻
  - 大屏/横屏：更多信息密度

### 色彩体系

| 角色 | 色值（建议） | 用途 |
|------|:----------:|------|
| 主色 | 🟠 橙色 `#FF6B35` | 品牌色、按钮、高亮、柴火icon |
| 辅助色 | 🟤 暖棕色 `#8B4513` | 次级文字、分割线、标签 |
| 背景 | ⚪ 米白 `#FFF8F0` | 页面背景（温暖感） |
| 强调 | 🔥 火焰红 `#FF4500` | 置顶标志、排行榜第一 |
| 暗黑 | ⚫ 暖黑 `#1A1A1A` | 暗黑模式背景 |

> 整套配色围绕「柴火」的温暖感展开，以橙色为绝对主色调

### 用户信息字段

|------|------|------|

| nickname | string | 昵称 |

| level | int | 用户等级 |

| invite_code | string | 邀请码 |
| invited_by | UUID | 邀请人ID |

### 馆主认证

馆主是真实经营场地的商家用户，通过认证后可获得场馆管理权限。

**申请材料：**
| 材料 | 说明 |
|------|------|
| 营业执照 | 企业/个体工商户营业执照照片 |
| 身份证正反面 | 法人/经营者身份证照片 |
| 联系方式 | 手机号 + 微信（用于运营联系） |
| 场地关联 | 关联到现有场地或多个场地 |

**认证流程：**
```
用户提交馆主申请
  → 上传营业执照
  → 上传身份证正反面
  → 填写联系方式
  → 选择关联场地
  → 后台人工审核
    ├── 通过 → 角色升级为「馆主」
    └── 不通过 → 告知原因，允许补充材料重提
```

**馆主权限：**
- 管理关联场地的信息、开放时间、收费设置
- 可设置场地为付费预定（Phase 2）
- 场地详情页显示「馆主认证」标识
- 可回复场地评价
- 拥有更高柴火提现额度（Phase 2）

### 馆主置顶广告系统

馆主可付费购买所在地区运动项目分类下的广告置顶位，将自己管理的场馆在主榜单中置顶展示，获取更多曝光。

#### 广告位规则

| 项目 | 说明 |
|------|------|
| 层级 | 五级：国家 / 省级 / 市级 / 区级 / 地级（街道） |
| 粒度 | 每一级 × 每个运动项目 × 一个置顶位 |
| 席位 | **每级每项目仅一席**（独占） |
| 时长 | 24小时，自动上下线 |
| 计费 | 以人民币购买，**非柴火** |
| 倒计时 | 置顶位显眼位置展示剩余时间 |

#### 价格体系

| 层级 | 价格（征求意见） |
|------|:-:|
| 地级（街道） | 66元 / 24小时 |
| 区级 | 待定 |
| 市级 | 待定 |
| 省级 | 待定 |
| 国家级 | 待定 |

#### 排队机制

```
馆主A购买「某市·篮球」置顶位
  → 检查当前是否有置顶中
    ├── 无人置顶 → 立即上线（计时24小时）
    └── 有人置顶 → 自动加入排队序列
         → 当前置顶到期后，系统自动激活下一个
         → 自动上线，无需人工操作
```

**排队队列逻辑：**
- 先购买→先排队（FIFO）
- 队列中的馆主可随时取消，按比例退款
- 到期前 App 推送通知提醒馆主
- 超过3次取消排入黑名单（限制7天）

#### 展示效果

- 置顶场地在榜单最上方，带「🔥置顶」角标
- 同时显示当前置顶馆主名称 + 倒计时
- 其他排队中馆主可见自己的排队序号和预计上线时间

#### 数据表

```
ad_placements       - 置顶广告投放记录
ad_queue            - 置顶排队队列
```

## 五、柴火经济体系

### 柴火获取

| 途径 | 数量 | 限制 |
|------|------|------|
| 每日登录 | 1根 | 每日一次 |
| 发布场地 | 100根 | 每场地一次（去重后） |
| 邀请好友 | 10根 | 每邀请一位好友完成注册 |
| 充值 | 1:1 | 1元 = 1根柴火 |

### 充值方案

**采用第三方个人支付平台**（XorPay / PayJS / 虎皮椒 / 易支付等）

**原因：** 66哥暂无公司/个体户，无法开通官方微信商户号。

**流程：**
```
用户APP发起充值
  → 后端调用第三方支付API生成付款码/支付链接
  → 用户扫码付款（个人微信收款码）
  → 第三方回调通知后端
  → 系统核验订单 → 自动发放柴火
```

**方案选型参考：**
| 平台 | 费率 | 特点 |
|------|:----:|------|
| XorPay | ~0.38% | 个人可注册，API完善 |
| PayJS | ~0.5% | 个人+个体户都行，文档全 |
| 虎皮椒 | ~0.5% | 支持APP支付 |
| 易支付 | 按平台 | 第三方聚合，费率各异 |

> 后期用户量大后，注册个体户营业执照 → 切换官方微信商户号（费率0.6%，更稳定）

### 柴火消费

| 途径 | 数量 | 说明 |
|------|------|------|
| 添柴（打赏场地） | 自定义 | 1根起，柴火转入场地总账 |

### 柴火 = 评分

场地的柴火总数直接影响该场地的评分/排名。
柴火越多 = 评分越高 = 排名越前。

### 防刷机制

- 同一物理场地仅第一个发布者获得100柴火
- 后续用户标记同一场地→进入「补充」模式，奖励少量或不奖励
- 去重检测：高德POI ID、50米围栏、名称相似度
- 后台可合并重复场地，柴火并入主场地

## 六、场地系统

### 发布流程

```
用户新建场地
  → 选择/拍摄照片（最多9张）
  → 拍摄视频（可选，30秒内）
  → 高德地图选点定位
  → 填写场地信息（名称、类型、描述、收费情况、开放时间）
  → 系统去重检测
    ├── 重复 → 进入补充模式 → 合并到原场地
    └── 唯一 → 发布成功 → +100柴火
```

### 场地字段

| 字段 | 类型 | 说明 |
|------|------|------|
| venue_id | UUID | 主键 |
| name | string | 场地名称 |
| type | string[] | 运动类型标签（篮球/足球/...） |
| description | text | 描述 |
| latitude | float | 纬度 |
| longitude | float | 经度 |
| poi_id | string | 高德POI ID（如有） |
| address | string | 地址 |
| photos | string[] | R2 图片URL列表 |
| videos | string[] | R2 视频URL列表 |
| publisher_id | UUID | 发布者 |
| is_free | boolean | 是否免费 |
| price_info | text | 收费说明 |
| open_hours | text | 开放时间 |
| chaihuo_total | int | 柴火总量（=评分） |
| status | enum | 待审核/已发布/已隐藏/已删除 |
| created_at | datetime | |

### 场地补充

其他用户可补充场地信息：
- 新增照片/视频
- 更新开放时间
- 补充收费信息
- 修正定位

每个补充获得少量奖励（5柴火或勋章经验），由审核通过后发放。

### 场地合并（后台）

管理员可合并重复场地：
- 合并后柴火总数相加
- 被合并场地的添柴记录归入主场地
- 原发布者的100柴火奖励保留

## 十一、举报/审核机制

### 举报内容

用户可举报：
- 违规场地（虚假、重复、不当内容）
- 违规评价/评论（广告、辱骂）
- 用户（恶意行为）

### 审核流程

```
用户举报
  → 系统记录，内容标记为"待审核"
  → 后台管理员/高等级用户审核
    ├── 通过 → 无操作
    └── 不通过 → 删除/隐藏内容，视严重程度处罚用户
```

### 处罚措施

| 违规次数 | 处罚 |
|----------|------|
| 1次 | 警告 |
| 2次 | 禁言7天 |
| 3次 | 冻结柴火30天 |
| 严重 | 永久封号 |

## 十二、广告位预留

### 广告位规划

| 广告位 | 位置 | 状态 |
|--------|------|:----:|
| 开屏广告 | APP启动时全屏展示 | 预留，暂不启用 |
| 页底广告 | 首页/榜单页底部 Banner | 预留，暂不启用 |
| 信息流广告 | 评论列表中部插入 | 预留，暂不启用 |

### 设计原则

- **UI预留：** 相应位置预留空白占位（灰色占位块），标注「广告位」
- **无广告时：** 占位块不显示/收窄，不干扰正常浏览
- **后台可编辑：** 管理员后台可独立控制每个广告位（启用/关闭/配置）
- **格式支持：** 图片、链接、跳转地址，后期可接入广告平台

### 后台广告管理字段

| 字段 | 说明 |

| link_url | 跳转链接 |
| duration | 展示时长（开屏广告专用） |

## 十三、俱乐部系统

### 概述

俱乐部是运动社交的核心组织单位，用户可创建/加入俱乐部，围绕共同运动爱好开展活动。

俱乐部可生成独立的分享页面，对外展示相当于一个「微网站」。

### 俱乐部类型

| 类型 | 说明 |
|------|------|
| 普通俱乐部 | 任意用户创建，自由加入/审核加入 |
| 认证俱乐部 | 管理员提交身份证认证，获得认证标识 |

### 基础信息

| 字段 | 说明 |
|------|------|
| 名称 | 俱乐部名称 |
| 头像 | 俱乐部Logo |
| 旗帜 | 俱乐部旗帜图片 |
| 介绍 | 俱乐部详细介绍 |
| 标语 | 简短的俱乐部口号 |
| 联系方式 | 微信 / 手机 / 邮箱 |
| 运动标签 | 支持多选（篮球/足球/羽毛球等） |
| 所在地区 | 省 / 市 / 区 |
| 创建时间 | 自动记录 |
| 成员数 | 实时统计 |
| 俱乐部柴火 | 成员添柴累积总量 |

### 成员管理

**角色体系：**

| 角色 | 权限 |
|------|------|
| 👑 创建者 | 全部权限（可转让） |
| 🛡 管理员 | 管理成员、发布公告、管理相册 |
| 👤 成员 | 参与活动、发布内容、邀请好友 |

**加入方式：**
- 公开加入：无需审核
- 审核加入：管理员审批
- 邀请加入：仅限被邀请用户

**管理操作：**
- 添加/移除管理员
- 踢出成员
- 成员退出俱乐部
- 转让创建者
- 解散俱乐部

### 俱乐部荣誉墙

- 内部约赛/约球战绩记录
- 获得的奖杯/勋章展示
- 俱乐部高光时刻（成员可标记）

### 俱乐部相册

- 创建者/管理员/成员可上传
- 活动合照、训练照片等
- 按时间线排列

### 俱乐部公告板

- 创建者/管理员可发布公告
- 公告类型：
  - 活动召集（约球/约赛）
  - 训练安排
  - 通知通告
- 成员可评论/点赞公告

### 俱乐部柴火

- 成员可将个人柴火添到俱乐部

#### D1 备份命令

# 手动触发备份
npx wrangler d1 backup create duichai-db

npx wrangler d1 backup list duichai-db

# 恢复备份
npx wrangler d1 backup restore duichai-db --backup-id=<id>

#### 自动化备份脚本（GitHub Actions）

```yaml
# .github/workflows/db-backup.yml
name: Database Backup
on:
  schedule:
    - cron: '0 2 * * *'   # 每天北京时间10点（UTC 2:00）
jobs:
  backup:
    runs-on: ubuntu-latest
    steps:
      - run: npx wrangler d1 backup create duichai-db
      - name: 导出并上传到 R2
        run: |
          npx wrangler d1 backup download duichai-db \
            --backup-id=$(latest) \
            --output=backups/$(date +%Y-%m-%d).sql
          # 上传到 R2 备份目录
```

## 二十、用户增长与运营策略

> 以下策略服务于两个核心目标：拉新（吸引更多运动爱好者）和留存（让用户持续活跃）。

### 一、游戏化玩法

#### 签到打卡进阶

```
基础：每日签到得1根柴火
├── 🏅 连续7天 → 额外+5根 + 「毅力柴」勋章
├── 🏅 连续30天 → 额外+30根 + 「老柴火」限定勋章
├── 🏅 连续100天 → 额外+100根 + 「千日柴」特殊徽章
└── 🏅 连续365天 → 终身成就称号「火炬大使」
```

**设计点：** 断签不归零，可用柴火补签（3根补一天），减少挫败感。

#### 场地探索任务

| 任务 | 奖励 |
|------|------|
| 打卡3个不同场地 | 20根柴火 |
| 打卡10个不同场地 | 100根柴火 + 「场地猎人」勋章 |
| 打卡本区全部场地 | 荣誉称号「区霸」 |
| 发布带视频的场地 | 额外+30根柴火 |
| 周五/周六发布 | 双倍柴火（周末效应） |

#### 柴火燃烧挑战

每月设置「燃烧挑战」：
- 本月累计添柴满100根 → 解锁限定头像框
- 本月累计添柴满1000根 → 全服通报 + 特殊称号

#### 开盲盒/宝箱

- 每次发布场地 → 随机开一个「柴火盲盒」（1-200根随机）
- 每次被添柴 → 有一定几率获得「幸运回火」（系统奖励1根）
- 首次登录每天转动一次「柴火轮盘」

### 二、社交裂变

#### 邀请机制升级

```
基础：邀请好友得10根柴火

升级玩法：
├── 📱 邀请排行榜 — 每月邀请TOP10全服展示 + 限定称号
├── 👫 邀请组队 — 和好友一起邀请，累计达标双方额外奖励
├── 📎 场地邀请码 — 分享场地时带你的邀请码，别人通过场地页注册也算你的
└── 🎁 邀请回流 — 被邀请人7天内活跃，邀请人额外得柴火
```

#### 俱乐部裂变

- 俱乐部可生成专属邀请海报（带二维码+俱乐部头像+标语）
- 创建俱乐部即得50根柴火启动资金
- 俱乐部满10人/50人/100人 → 创建者获得对应勋章
- 俱乐部可发起「拉新擂台」— 哪个俱乐部周增成员最多，全俱乐部发福利

#### 场地分享链

- 分享场地到微信 → 点进来的人看到「来自XX的推荐」
- 通过分享链接注册 → 分享者 + 被分享者各得5根
- 场地分享页面自带热度数据（当前柴火数+人气）

---

### 三、内容驱动

#### 热力图/活动地图

- 首页地图模式显示「当前热度」— 暖色到冷色表示附近场地活跃度
- 红色=高热度 → 吸引用户去探索

#### 运动日记/Timeline

- 用户的运动足迹地图（去过哪些场地）
- 每次评论/添柴/发布 → 自动生成Timeline
- 年底生成「年度运动报告」：你去了几个场地、添了多少柴
- 可分享到朋友圈（刷屏级传播）

#### 高质量内容推荐

- 精选场地推荐（编辑推荐/算法推荐）
- 「本周最火场地」「新晋场地」「冷门宝藏场地」
- 视频优先，双列瀑布流 + 视频自动播放

#### 运动百科/UGC指南

- 用户可写场地攻略/运动技巧（图文）
- 优质攻略可推上首页，获得柴火打赏
- 类似小红书+知乎的内容形式

### 七、增长飞轮

```
用户发布场地
  → 获得柴火（正向激励）
  → 场地吸引他人添柴/评论
  → 场地热度上升 → 上榜
  → 更多人看到 → 注册/下载
  → 新用户发布更多场地
  └── 循环 🔄
```

```
用户加入俱乐部
  → 参与活动 → 贡献柴火
  → 俱乐部排名上升
  → 吸引更多人加入
  └── 循环 🔄
```
